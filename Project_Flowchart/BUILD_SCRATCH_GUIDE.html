📡 Runtime Code Flow — What Happens When You Type a Query

Step-by-step trace: "What's the budget burn rate?"

1. FRONTEND frontend/app.py User types query → st.chat_input() captures text → call_backend(query) sends POST /chat → requests.post("http://localhost:8000/chat", json={query, session_id}) ↓ HTTP POST (JSON) 2. BACKEND backend/main.py FastAPI receives request → Pydantic validates ChatRequest → run_chat(query, session_id) called → passes control to the router ↓ Python function call 3. ROUTER router/graph.py 3a. build_graph() creates LangGraph StateGraph (once, cached) 3b. route_query(state) — 3-tier classification: Tier 1: scan keywords → "budget" found in API_KEYWORDS ✅ → returns "api" 3c. LangGraph routes to api_node() ↓ Conditional edge in graph 4. API AGENT agents/api_agent.py 4a. _extract_metric_intent(query) → calls Groq LLM (temp=0) → LLM returns "budget_burn_rate" 4b. Route to SQL connector: sql_connector.get_metric("budget_burn_rate") → returns {value: 91, unit: "%", status: "High"} 4c. _format_response(data) → calls Groq LLM (temp=0.3) → LLM formats: "Budget burn rate is 91%. Status: High." ↓ Return dict 5. CONNECTOR connectors/sql_connector.py 5a. Check os.getenv("SQL_SERVER") → Not set → use MOCK_KPI_DATA dict 5b. Return {"budget_burn_rate": {value:91, unit:"%", status:"High"}} ↓ Dict flows back up 6. BACKEND backend/main.py Receives {answer, agent_used, sources} → Wraps in ChatResponse (Pydantic model) → Returns JSON HTTP 200 ↓ HTTP Response (JSON) 7. FRONTEND frontend/app.py Receives response → displays: → Agent badge: 📊 API Agent (yellow) → Answer text: "Budget burn rate is 91%..." → Sources: (none for API Agent) → Saves to st.session_state.messages

File execution order for EACH query

# File Function Called What It Does
1 frontend/app.py call_backend(query) HTTP POST to /chat
2 backend/main.py chat_endpoint() Pydantic validation → calls run_chat()
3 router/graph.py run_chat() → route_query() 3-tier routing → picks agent
4 router/graph.py *_node() (e.g. api_node) LangGraph node → calls agent
5 agents/*.py run_*_agent() LLM extraction → connector call → LLM format
6 connectors/*.py get_metric() / create_ticket() etc. Try real API → fallback to mock
7 backend/main.py return ChatResponse JSON response to frontend
8 frontend/app.py st.markdown(answer) Display answer + badge

Which agent calls which files?

Agent Calls These Connectors Uses LLM For

Create ingestion/embedder.py

Purpose: Convert text chunks into 384-dim vectors using HuggingFace model.

from sentence_transformers import SentenceTransformer

_model = None  # Singleton — loaded once

def get_model():
    global _model
    if _model is None:
        _model = SentenceTransformer("all-MiniLM-L6-v2")  # ~80MB download
    return _model

def embed_chunks(chunks) -> np.ndarray:
    texts = [c["text"] for c in chunks]
    return get_model().encode(texts)  # shape (N, 384)

def embed_query(query) -> np.ndarray:
    return get_model().encode([query])  # shape (1, 384)

Singleton pattern: Loading the model takes ~2s. We load ONCE and reuse across all requests.

Step 5 — Router (LangGraph StateGraph)

Create router/graph.py — the orchestrator

# 1. Define typed state
class ChatState(TypedDict):
    query: str
    session_id: str
    agent_used: str
    answer: str
    sources: list

# 2. Define keyword lists
HELPDESK_KEYWORDS = ["ticket", "raise", "broken", "laptop", ...]
WORKFLOW_KEYWORDS = ["email", "approve", "raid", "notify", ...]
API_KEYWORDS      = ["kpi", "metric", "budget", "sprint", ...]
RETRIEVAL_KEYWORDS= ["document", "sop", "policy", "report", ...]

# 3. Define anchor sentences for Tier 2
AGENT_ANCHORS = {
    "retrieval": ["Find information in project documents", ...],
    "api":       ["Show me KPI metrics and project data", ...],
    ...
}

# 4. Pre-embed anchors at module load
ANCHOR_EMBEDDINGS = {agent: model.encode(sents) for agent, sents in ...}

# 5. Route function (3 tiers)
def route_query(state):
    query = state["query"].lower()
    # Tier 1: keyword scan
    for word in HELPDESK_KEYWORDS:
        if word in query: return "helpdesk"
    ...
    # Tier 2: cosine similarity
    q_vec = model.encode([query])
    for agent, anchors in ANCHOR_EMBEDDINGS.items():
        score = cosine_sim(q_vec, anchors).max()
        if score > 0.35: return agent
    # Tier 3: Groq LLM
    return groq_classify(query)

# 6. Build LangGraph
def build_graph():
    g = StateGraph(ChatState)
    g.add_node("retrieval", retrieval_node)
    g.add_node("api", api_node)
    g.add_node("helpdesk", helpdesk_node)
    g.add_node("workflow", workflow_node)
    g.add_conditional_edges(START, route_query, {...})
    g.add_edge("retrieval", END)
    # ... same for other nodes
    return g.compile()

# 7. Entry point for backend
def run_chat(query, session_id=None):
    graph = get_graph()  # singleton
    return graph.invoke({"query": query, "session_id": session_id, ...})

Step 6 — Backend (FastAPI REST API)

Create backend/main.py

from fastapi import FastAPI, UploadFile, File, Form
from pydantic import BaseModel

app = FastAPI(title="KPMG PMO AI Chatbot", version="2.0.0")

# CORS — allow Streamlit (8501) to call FastAPI (8000)
app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)

class ChatRequest(BaseModel):
    query: str
    session_id: str = None

class ChatResponse(BaseModel):
    answer: str
    agent_used: str
    sources: list

@app.post("/chat")
def chat(request: ChatRequest) -> ChatResponse:
    result = run_chat(request.query, request.session_id)
    return ChatResponse(**result)

@app.post("/upload")            # File upload
@app.post("/upload/clear")      # Clear session uploads
@app.get("/upload/files")       # List uploaded files
@app.get("/health")             # Health check
@app.get("/tickets")            # Admin: view tickets
@app.get("/logs")               # Admin: view RAID logs
@app.get("/approvals")          # Admin: view approvals
