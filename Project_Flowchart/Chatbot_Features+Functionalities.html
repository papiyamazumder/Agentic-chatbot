How each capability is technically built — 13 components
T01
🔀
Multi-Agent Orchestration
LangGraph StateGraph routing
LangGraph orchestrates 3 specialised agents using a StateGraph. route_query() classifies each query by keyword matching and directs to Retrieval, API, or Workflow agent. Fully modular — agents have single responsibility with no overlap.
LangGraphStateGraphConditional routing
T02
🔍
RAG Pipeline
Retrieval-Augmented Generation
Documents chunked at 800 chars with 100-char overlap using RecursiveCharacterTextSplitter. Each chunk embedded via text-embedding-3-small into 1536-dim vectors. Stored in FAISS (P1) or FAISS + BM25 (P2). Top-5 chunks injected as LLM context.
LangChainFAISS / AI SearchTop-k retrieval
T03
🧮
Semantic Vector Search
Dense vectors — not keyword matching
Query embedded at runtime → cosine similarity finds semantically relevant chunks even when exact keywords don't match. Phase 2 upgrades this with FAISS + BM25 semantic re-ranking layer on top of vector similarity.
1536-dim vectorsCosine similaritySemantic re-rank (P2)
T04
🧠
LLM Intent Classification
NLP-based action extraction
Workflow Agent uses an LLM prompt to parse natural language into structured JSON — extracting action type (create_ticket / send_email / approve / update_log) and all parameters. No rigid command syntax required from the user.
Prompt engineeringJSON extractionLLaMA 3.1 8B
T05
🔗
REST API Integration
Live KPI data via HTTP endpoints
API Agent calls internal REST endpoints with Bearer auth headers. LLM extracts metric name → requests.get() fetches data → LLM formats JSON response into readable output. SQLAlchemy MySQL fallback if API unavailable.
requestsSQLAlchemyBearer authMySQL fallback
T06
⚡
Workflow Automation Engine
Python functions → Logic Apps
Phase 1: smtplib email, in-memory ticket dicts, local JSON logs. Phase 2: Workflow Agent POSTs to Azure Logic App HTTP trigger URL → Logic Apps handle real Outlook email, ITSM tickets, Teams approvals, RAID log updates. Zero agent code change between phases.
smtplib (P1)Logic Apps (P2)HTTP triggers
T07
🚀
FastAPI Backend
REST API with Pydantic + Swagger
POST /chat endpoint receives queries. Pydantic models validate request/response schema. Swagger UI auto-documents the API. Postman used for testing. Async uvicorn ASGI server. Deployed as Docker container on Azure App Service in Phase 2.
FastAPIPydanticSwagger UIuvicorn
T08
📄
Document Ingestion Pipeline
Decoupled — automated in Phase 2
Separate pipeline: PyMuPDF loads PDFs, python-docx for Word files, pandas for Excel. RecursiveCharacterTextSplitter chunks text. OpenAI embeds. FAISS/AI Search indexes. Decoupled from chatbot. Phase 1: manual run. Phase 2: Azure Functions nightly timer trigger.
PyMuPDFpython-docxNightly schedule (P2)
T09
🖥️
Streamlit Conversational UI
Chat interface with session state
st.chat_message and st.chat_input render chat bubbles. Conversation history stored in st.session_state as list of role/content dicts. Each response shows agent badge. HTTP POST to FastAPI. Deployed on Azure App Service via Docker in Phase 2.
Streamlitsession_stateDocker (P2)
T10
🔐
Enterprise Security
.env → Azure Key Vault
Phase 1: python-dotenv loads secrets from .env file into os.environ. Phase 2: Azure Key Vault stores all API keys and connection strings. App uses Managed Identity — no credentials in code. Secret rotation automatic. Full audit log. .env kept only for local dev.
python-dotenv (P1)Key Vault (P2)Managed Identity
T11
🔐
RBAC & Session Auth
Role-based access via session state
3 roles: PMO Admin (level 3), Manager (level 2), Resource (level 1). Login stores role in st.session_state. call_backend() checks allowed agents per role — blocks unauthorized agent responses with access denied message.
RBACsession_stateRole filtering
T12
👤
Onboarding/Offboarding Engine
Outlook Graph API + approval workflow
LLM extracts {action, employee_name, role, project} → outlook_connector creates record + checklist → send_approval_notification() sends email via Microsoft Graph API /sendMail with OAuth2 client credentials. Mock fallback logs locally.
Graph APIOAuth2LLM JSON extraction
T13
👥
User Management Panel
Admin-only role grant/revoke
PMO Admin tab with selectbox per user to change roles. Updates USERS dict + managed_users session state. Changes take effect on next login. Includes access level reference table.
Admin panelSelectboxSession persistence

Logic Apps approval chain
LLM extracts action: approve + approver + details → POST to Logic App URL → Logic App sends adaptive approval card to Teams/Outlook → waits for response → updates RAID log on completion. Full audit trail.

route_query() lowercases query → checks retrieval_keywords list first → then api_keywords → then workflow_keywords → returns "retrieval" / "api" / "workflow" → LangGraph routes to node → default fallback: retrieval. set_conditional_entry_point(route_query, mapping).
