🎯 30-Second Elevator Pitch

I built a multi-agent RAG chatbot for KPMG's PMO team that handles 4 types of tasks through specialized AI agents: document Q&A (RAG over SharePoint/Teams docs), live KPI queries (SQL + Jira), IT helpdesk automation (ServiceNow tickets), and workflow actions (Outlook email, Teams approvals, RAID logs, onboarding/offboarding, resource tagging). The system uses a 3-tier hybrid router (keyword → embedding similarity → LLM) built with LangGraph, and includes RBAC with 3 roles (PMO Admin, Manager, Resource), user management, and manager approval notifications via Microsoft Graph API.

💡 Key Technical Concepts

# Concept How We Use It
1 RAG Embed query → FAISS top-5 → inject as context → LLM generates grounded answer
2 Multi-Agent Architecture 4 specialized agents, each with own tools/prompts, orchestrated by LangGraph
3 LangGraph StateGraph Typed state dict flows through conditional edges to the right agent node
4 3-Tier Hybrid Routing Keyword → Embedding similarity → LLM fallback (free, fast, accurate)
5 Sentence Transformers all-MiniLM-L6-v2 for 384-dim embeddings (routing + retrieval)
6 FAISS Vector Search IndexFlatL2 for brute-force exact similarity search
7 Temperature Control 0 for extraction, 0.2 for factual, 0.3 for formatting
8 Singleton Pattern LLM client, embedding model, FAISS index — loaded once, reused
9 Mock Fallback Every connector: try real API, catch exception, return mock
10 Pydantic Validation ChatRequest/ChatResponse enforce schema at API boundary
11 RBAC 3 roles enforced at frontend + backend
12 Session-Scoped Storage Uploaded files per session_id, cleared on end
13 Approval Workflow Onboarding/offboarding trigger manager notifications via Graph API
