Layer 4 — The 3 Agents
🔍 Retrieval Agent
InputAny query about PDFs, SOPs, meeting notes, reports
Step 1 — Embed queryCalls OpenAI text-embedding-3-small to vectorise the user's question
Step 2 — Search FAISSFinds top-5 most similar chunks from local FAISS index using cosine similarity
Step 3 — Build promptCombines retrieved chunks as context + original question into a structured prompt
Step 4 — Call LLMGPT-4o generates answer using only the retrieved context
OutputNatural language answer with source document names
FAISS searchOpenAI GPT-4o
📊 API Agent
InputAny query about KPIs, metrics, project performance, dashboards
Step 1 — Extract intentLLM reads the query and extracts which KPI or metric the user wants
Step 2 — Call REST APIPython requests.get() calls the internal API endpoint for that KPI with auth token
Step 3 — Fallback to DBIf API fails, SQLAlchemy queries MySQL directly for the metric
Step 4 — Format outputLLM formats raw JSON/SQL result into executive-friendly Power BI-ready summary
OutputFormatted KPI data ready for reporting
REST APIMySQLpandas
🔄 Workflow Agent
InputTicket creation, approval requests, email notifications, log updates
Step 1 — Classify actionLLM reads query → returns JSON with action type and details
Step 2 — Execute actionPython function runs: create_ticket / send_email / approve / update_log
Ticket creationGenerates ticket ID, stores in in-memory dict / local JSON log
Email notificationsmtplib sends email directly from Python
Log updateAppends entry to local system_logs list or text file
OutputConfirmation message with action result
smtpliblocal logs

The Core Insight for Your Interview
Every agent, every prompt, all LangGraph routing, the entire FastAPI API, and the Streamlit UI are identical in both phases. Azure doesn't change what the system does — it changes where it runs and how it connects to infrastructure. The AI intelligence is 100% yours. Azure is the enterprise-grade plumbing underneath it.
