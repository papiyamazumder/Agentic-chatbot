Layer 2 — Backend Orchestration
⚙️
FastAPI + LangGraph Router
Azure App Service (API Backend) or Azure Container Apps
The brain of the system. Receives user queries, decides which agent to route to (Retrieval / API / Workflow) using LangGraph. Manages session state and response assembly.

↓

Layer 3 — Multi-Agent Layer
🔍 Retrieval Agent
Purpose Unstructured doc queries — PDFs, SOPs, Meeting Notes, Reports
Deployed via Logic inside FastAPI (same App Service)
Calls → Azure AI Search (vector search) + Azure OpenAI (LLM answer)
📊 API Agent
Purpose KPI data retrieval → feeds Power BI dashboards
Deployed via Logic inside FastAPI + Azure Functions (for scheduled pulls)
Calls → REST APIs / internal systems + Azure OpenAI for formatting
🔄 Workflow Agent
Purpose Ticket creation, approvals, email notifications, log updates, onboarding/offboarding, resource tagging
Deployed via Azure Logic Apps (no-code triggers) + Azure Functions (custom code)
Calls → External systems: ITSM, Email (Graph API), RAID logs, Onboarding records, Copilot Studio
↓
Each agent calls its Azure service
↓
