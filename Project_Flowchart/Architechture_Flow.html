Layer 3 — Multi-Agent Layer
🔍 Retrieval Agent
Purpose Document Q&A — PDFs, SOPs, meeting notes via hybrid RAG
Pipeline FAISS (cosine) + BM25 (keyword) → RRF fusion → Cross-Encoder rerank
Calls → Local FAISS index + Groq LLaMA 3.1 for answer generation
📊 API Agent
Purpose KPI data retrieval from Excel trackers and REST APIs
Pipeline LLM extracts metric intent → kpi_file_connector reads data → LLM formats
Calls → kpi_file_connector + sql_connector + jira_connector + Groq LLM
🎫 Helpdesk Agent
Purpose IT ticket creation, status checks, incident management
Pipeline LLM classifies action → ServiceNow connector creates/queries tickets
Calls → servicenow_connector + Groq LLM for field extraction
⚙️ Workflow Agent
Purpose Emails, approvals, RAID logs, onboard/offboard/tag resources
Pipeline LLM classifies 8 sub-actions → routes to appropriate connector
Calls → outlook + teams + sharepoint + azure_insights connectors
↓
Each agent calls its Azure service

Layer 4 — Azure Cloud Services

Hybrid Vector + Keyword Index
FAISS IndexFlatIP (384-dim, cosine) for semantic search + BM25Okapi for keyword matching. Results fused via Reciprocal Rank Fusion, re-ranked by Cross-Encoder.
